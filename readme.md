Django 是一个用 Python 编写的高级 Web 框架，它鼓励快速开发和干净、实用的设计。

### 1. Django 的核心概念

- **MVC/MTV 模式**：Django 遵循模型（Model）- 模板（Template）- 视图（View）架构（有时也称作
  MTV，即模型-模板-视图），这有助于开发者将应用的不同部分分离，使得代码更加清晰、易于维护。
    - **模型（Model）**：负责数据的存取，与数据库表对应。
    - **模板（Template）**：用于生成发送给用户的 HTML 网页。Django 提供了强大的模板系统，允许你轻松地控制网页的展示。
    - **视图（View）**：处理业务逻辑，接收 Web 请求，进行数据处理，并调用模板渲染页面或返回 JSON 数据等。

- **ORM（对象关系映射）**：Django 的 ORM 允许你以 Python 类的形式定义数据库模型，并通过 Python 代码来操作数据库，无需编写 SQL
  语句。

- **URL 分发**：Django 有一个 URLconf 机制，用于将 URL 映射到视图函数。这使得你可以根据 URL 路径的不同来执行不同的逻辑。

- **Admin 界面**：Django 自动生成一个管理界面，让你无需编写任何代码就能管理网站的内容。这对于内容管理型网站尤其有用。

### 2. Django 的优势

- **快速开发**：Django 提供了大量的内建功能，如用户认证、内容管理、会话管理、表单处理等，极大地加速了开发过程。
- **安全性**：Django 提供了许多安全相关的功能，如 SQL 注入防护、跨站脚本防护（XSS）、跨站请求伪造防护（CSRF）等。
- **可扩展性**：Django 框架设计灵活，易于扩展，你可以通过第三方库和插件来增加新的功能。

### 3. 如何快速开始

1. **安装 Python 和 Django**：
    - 确保你的系统中安装了 Python（推荐 Python 3.6 及以上版本）。
    - 使用 pip 安装 Django：`pip install django`

2. **创建项目**：
    - 打开命令行或终端，运行 `django-admin startproject myproject` 来创建一个名为 `myproject` 的 Django 项目。

3. **创建应用**：
    - 进入项目目录，运行 `python manage.py startapp myapp` 来创建一个应用。

4. **定义模型**：
    - 在应用的 `models.py` 文件中定义你的数据模型。

5. **迁移数据库**：
    - 运行 `python manage.py makemigrations` 和 `python manage.py migrate` 来创建数据库表。

6. **创建视图**：
    - 在应用的 `views.py` 文件中编写视图逻辑。

7. **配置 URL**：
    - 在项目的 `urls.py` 文件中或应用的 `urls.py`（如果创建了）中配置 URL 分发。

8. **开发模板**：
    - 在应用的 `templates` 目录下创建 HTML 文件。

9. **运行开发服务器**：
    - 运行 `python manage.py runserver` 启动开发服务器，并在浏览器中访问 `http://127.0.0.1:8000/` 来查看你的应用。

### 什么是HTML？

HTML是构建网页的基石，它不是一种编程语言，而是一种标记语言。你可以把它想象成一种告诉浏览器如何显示网页内容的“指令集”。通过使用HTML，你可以创建文本、链接、图片、列表、表格等元素，并将它们组织成网页。

### 为什么学习HTML？

学习HTML是成为网页设计师或前端开发者的第一步。掌握HTML后，你可以开始构建基本的网页，并进而学习CSS（层叠样式表）来美化网页，以及JavaScript来添加交互性。

### 如何开始学习HTML？

1. **了解基本结构**：
    - 每个HTML页面都以`<html>`标签开始和结束。
    - `<head>`部分包含了文档的元数据，如标题（`<title>`）、字符集定义（`<meta charset="utf-8">`
      ）和链接到CSS文件（`<link rel="stylesheet" href="style.css">`）。
    - `<body>`部分包含了网页的可见内容，如文本、图片、链接等。

2. **学习基本标签**：
   在HTML（HyperText Markup Language）中，标签是用来定义网页内容结构的元素。它们被成对的尖括号`<`和`>`
   包围，通常成对出现（开始标签和结束标签），但也有一些是自闭合的。标签告诉浏览器如何展示内容，以及内容之间的结构关系。下面是一些常见的HTML标签介绍：

### 1. 结构性标签

- **`<html>`**：整个HTML文档的根元素，包含了`<head>`和`<body>`两部分。
- **`<head>`**：包含了文档的元数据，如标题、字符集定义、链接到CSS和JavaScript文件等。
- **`<body>`**：包含了网页的可见内容，如文本、图片、视频、音频、游戏、表单等。

### 2. 文本格式化标签

- **`<h1>`到`<h6>`**：六级标题，`<h1>`是最高级别的标题，通常用于主标题。
- **`<p>`**：段落。
- **`<br>`**：换行，是一个自闭合标签。
- **`<strong>`** 和 **`<b>`**：加粗文本，但`<strong>`表示内容的重要性，而`<b>`仅表示样式上的加粗。
- **`<em>`** 和 **`<i>`**：斜体文本，`<em>`表示内容的强调，而`<i>`仅表示样式上的斜体。

### 3. 链接和图片

- **`<a>`**：超链接，用于从一个页面链接到另一个页面或页面内的某个部分。
- **`<img>`**：图片，用于在网页中嵌入图片，通过`src`属性指定图片的路径，`alt`属性提供图片的文字描述。

### 4. 列表

- **`<ul>`**：无序列表，列表项使用`<li>`标签定义。
- **`<ol>`**：有序列表，同样使用`<li>`标签定义列表项，但列表项会带有数字或字母标记。
- **`<dl>`**：定义列表，常用于术语定义，包含`<dt>`（定义项）和`<dd>`（定义描述）。

###  

- **`<tr>`**：定义表格中的一行。
- **`<th>`**：定义表格的表头单元格，通常包含标题或列标题。
- **`<td>`**：定义表格的标准单元格，包含数据。

### 6. 表单

- **`<form>`**：用于创建HTML表单，收集用户输入。
- **`<input>`**：用于收集用户输入，可以是文本、密码、单选按钮、复选框等。
- **`<label>`**：定义`<input>`元素的标签，提高表单的可访问性。
- **`<button>`**：定义可点击的按钮，常用于提交表单。
- **`<select>`**：创建下拉选择菜单。
- **`<option>`**：定义`<select>`元素中的选项。

### 7. 其他常用标签

- **`<div>`** 和 **`<span>`**：`<div>`是块级元素，常用于布局；`<span>`是行内元素，常用于对文档中的小块内容应用样式或脚本。
- **`<meta>`**：位于`<head>`部分，提供关于HTML文档的元数据，如字符集、页面描述、关键词等。
- **`<title>`**：定义文档的标题，显示在浏览器的标题栏或页面的标签上。

这些只是HTML中众多标签的一小部分，但它们构成了网页结构的基础。通过组合这些标签，你可以创建出丰富多彩、功能强大的网页。
